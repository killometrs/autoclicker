## 📖 Описание

**Safe Enterprise Key & Mouse Catcher** - это мощная программа для автоматизации ввода клавиш и кликов мыши. Она позволяет создавать последовательности действий с настраиваемыми задержками, повторениями и поддержкой различных методов ввода, включая специальный режим для игр.

### 🔥 Ключевые особенности

- ✅ Полная симуляция нажатий клавиш и кликов мыши
- 🎮 Специальный игровой режим с поддержкой DirectInput
- 🔄 Настраиваемые повторения и циклы
- 💾 Автосохранение и загрузка настроек (JSON)
- ⌨ Глобальные хоткеи для управления
- 🎯 Поддержка комбинаций клавиш (Ctrl+C, Alt+Tab и т.д.)
- ⚡ Работает в играх на Unity, Unreal Engine и других
- 🌐 Поддержка Unicode (любые языки и символы)
- 🔒 Работает в фоновом режиме
- 📊 Прогресс-бар выполнения
- 🎨 Красивый и интуитивный интерфейс

## 🎯 Основные возможности

### Поддерживаемые действия

| Тип | Описание | Примеры |
|-----|----------|---------|
| **Текст** | Печать любого текста | `Привет`, `Hello`, `123` |
| **Клавиши** | Нажатие отдельных клавиш | `A`, `1`, `F5`, `Пробел` |
| **Комбинации** | Сочетания клавиш | `Ctrl+C`, `Alt+Tab`, `Shift+W` |

| Параметр | Описание | Пример |

| 🎲 **Макс (мс)** | Максимальный случайный разброс | `500` |
| 🔁 **Повторов** | Сколько раз выполнить действие | `3` |
| 🎮 **Метод ввода** | Способ симуляции | `SCANCODE` |

## 🎮 Использование

### 1. Добавление действия

#### Через кнопку "Задать" (рекомендуется)
1. Нажмите кнопку **"Задать"**
2. Нажмите нужную клавишу или кликните мышью
3. В поле появится название действия

#### Ручной ввод
```
w              # Клавиша W
space          # Пробел
enter          # Enter
Ctrl+C         # Комбинация клавиш
Привет         # Текст "Привет"
Клик ЛКМ       # Клик левой кнопкой мыши
```

### 2. Настройка параметров

| Параметр | Описание | Пример |
|----------|----------|---------|
| **Задержка** | Время между действиями (сек) | `1.0` |
| **Мин (мс)** | Минимальный случайный разброс | `0` |
| **Макс (мс)** | Максимальный случайный разброс | `500` |
| **Повторов** | Количество повторений действия | `3` |

### 3. Управление последовательностью

- **Добавить** - добавляет действие в список
- **Удалить** - удаляет выбранное действие
- **Очистить всё** - удаляет все действия
- **Сохранить** - сохраняет настройки в файл
- **Загрузить** - загружает настройки из файла

### 4. Запуск

Нажмите **"СТАРТ"** или используйте глобальный хоткей (по умолчанию `Ctrl+Z`)

## 🎮 Игровой режим

### Что это?
Игровой режим оптимизирует ввод для игр, используя специальные методы симуляции нажатий, которые воспринимаются игровыми движками.

### Как включить
1. Отметьте чекбокс **"🎮 Игровой режим"**
2. Выберите метод ввода:
   - `SCANCODE` - рекомендуется для большинства игр
   - `DIRECT_INPUT` - для игр на Unity/Unreal
   - `SENDINPUT` - стандартный метод
   - `KEYBD_EVENT` - для старых игр

### Как это работает в играх
```
Программа → Отправляет скан-код клавиши → Игра получает событие нажатия
```

Пример:
```
Действие: "W" → Отправляется скан-код 0x11 → Игра воспринимает как W
Действие: "Space" → Отправляется скан-код 0x39 → Игра воспринимает как пробел
```

## 📊 Методы ввода

| Метод | Описание | Когда использовать |
|-------|----------|-------------------|
| **SENDINPUT** | Стандартный метод Windows | Обычные приложения |
| **SCANCODE** | Отправка скан-кодов | **Игры (рекомендуется)** |
| **DIRECT_INPUT** | DirectX API | Игры на Unity/Unreal |
| **KEYBD_EVENT** | Старый метод keybd_event | Старые игры |

- Изменении хоткея
- Изменении параметров
- Закрытии программы

### Формат файла (JSON)
```json
{
  "version": "1.2",
  "items": [
    {
      "type": "Текст",
      "type_enum": "TEXT",
      "display_text": "Привет",
      "action_text": "Привет",
      "delay": 1.0,
      "rand_min": 0,
      "rand_max": 500,
      "repeat_count": 3,

    "mods": ["Ctrl"],

  "total_cycles": 5,
  "game_mode": true,
  "input_method": "SCANCODE"
}
```

## ❓ Часто задаваемые вопросы

### Q: Почему программа не работает?
**A:** Проверьте:
1. Запущена ли программа от имени администратора
2. Правильно ли установлены зависимости
3. Активно ли целевое окно

### Q: Как работать с игрой?
**A:** 
1. Включите игровой режим
2. Выберите `SCANCODE` как метод ввода
3. Запустите последовательность
4. Переключитесь в игру

### Q: Почему игра не реагирует на нажатия?
**A:** Попробуйте:
1. Разные методы ввода (`SCANCODE`, `DIRECT_INPUT`)
2. Увеличьте задержки между действиями
3. Убедитесь, что игра в фокусе

### Q: Как остановить автоматизацию?
**A:** 
- Нажмите кнопку **"СТОП"**
- Используйте глобальный хоткей (по умолчанию `Ctrl+Z`)
- Закройте программу

### Q: Безопасно ли использовать в играх?
**A:** Программа эмулирует обычные нажатия клавиш, но использование в онлайн-играх может нарушать правила игры. Используйте на свой риск.

## 🛠 Решение проблем

### Проблема: "NameError: name 'KEYBDINPUT' is not defined"
**Решение:** Убедитесь, что структуры ctypes определены на уровне модуля.

### Проблема: Хоткей не работает
**Решение:** Запустите программу от имени администратора.

### Проблема: В игре не работает ввод
**Решение:**
1. Включите игровой режим
2. Выберите метод `SCANCODE`
3. Попробуйте увеличить задержки

### Проблема: Не сохраняются настройки
**Решение:** Проверьте права на запись в папку с программой.

### Проблема: Ошибка импорта библиотек
**Решение:** 
```bash
pip install --upgrade keyboard pywin32 psutil
```
